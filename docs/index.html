Overview

TDB is a full-featured SQL database engine written in C11 and C++20. It supports 53 statement types, 200+ scalar functions, PL/SQL stored procedures, Lua and JavaScript scripting with optional LLVM JIT compilation, AES-256-GCM encryption, partitioned tables, window functions, recursive CTEs, cursors, and graph queries.

Advanced SQL

Window functions, recursive CTEs, GROUPING SETS/CUBE/ROLLUP, ON CONFLICT upsert, RETURNING, MERGE, graph MATCH.

Spatial Support

40+ PostGIS-compatible functions, R-Tree and R+Tree indexes, WKT geometry, distance calculations.

Quick Start

Build

mkdir build && cd build
cmake .. -DTDB_BUILD_TESTS=OFF
cmake --build . --target tdbcli --target tdb-tui -j8

Run

# Interactive mode
./tdbcli

# Named database (auto-saves)
./tdbcli mydb.tdb

# Execute SQL file
./tdbcli -f examples/erp_sample.sql

# Single statement
./tdbcli -e "SELECT 1 + 1 AS result"

DDL Statements

CREATE [TEMPORARY] TABLE [IF NOT EXISTS] name (
    col INTEGER PRIMARY KEY AUTO_INCREMENT,
    name VARCHAR(100) NOT NULL,
    email VARCHAR(255) UNIQUE,
    salary DECIMAL(10,2) CHECK (salary > 0),
    dept_id INTEGER REFERENCES departments(id) ON DELETE CASCADE,
    created TIMESTAMP DEFAULT CURRENT_TIMESTAMP
) ENCRYPTED
  PARTITION BY RANGE (created) (
    PARTITION p1 VALUES LESS THAN ('2026-01-01'),
    PARTITION p2 VALUES LESS THAN MAXVALUE
);

All CREATE statements support IF NOT EXISTS. All DROP statements support IF EXISTS and CASCADE.

DML Statements

SELECT

[WITH [RECURSIVE] cte AS (SELECT ...)]
SELECT [DISTINCT] expr [AS alias], ...
FROM table [INNER|LEFT|RIGHT JOIN table ON cond]
[WHERE condition]
[GROUP BY expr | GROUPING SETS(...) | CUBE(...) | ROLLUP(...)]
[HAVING condition]
[ORDER BY expr [ASC|DESC] [NULLS FIRST|LAST]]
[LIMIT n] [OFFSET n]

INSERT with Upsert

INSERT INTO users (name, email) VALUES ('Alice', 'a@b.com')
ON CONFLICT DO UPDATE SET name = 'Alice Updated'
RETURNING id, name;

UPDATE / DELETE with RETURNING

UPDATE users SET active = FALSE WHERE last_login < '2025-01-01' RETURNING id, name;
DELETE FROM sessions WHERE expired = TRUE RETURNING user_id;

MERGE

MERGE INTO target USING source ON target.id = source.id
WHEN MATCHED THEN UPDATE SET val = source.val
WHEN NOT MATCHED THEN INSERT (id, val) VALUES (source.id, source.val);

Window Functions 15

SELECT name, dept, salary,
    ROW_NUMBER() OVER (ORDER BY salary DESC) AS rn,
    RANK() OVER (PARTITION BY dept ORDER BY salary DESC) AS dept_rank,
    LAG(salary) OVER (ORDER BY salary) AS prev_sal,
    SUM(salary) OVER (ORDER BY salary ROWS UNBOUNDED PRECEDING) AS running
FROM employees;

Ranking: ROW_NUMBER, RANK, DENSE_RANK, NTILE, PERCENT_RANK, CUME_DIST

Value: LAG, LEAD, FIRST_VALUE, LAST_VALUE, NTH_VALUE

Aggregate: SUM, COUNT, AVG, MIN, MAX as running/sliding window aggregates

PL/SQL

CREATE PROCEDURE factorial(n INT) AS
    'DECLARE result INT := 1;
     DECLARE i INT := 1;
     WHILE i <= n LOOP
         result := result * i;
         i := i + 1;
     END LOOP;
     RETURN result';

CALL factorial(5);  -- Returns 120

Control Flow

  • DECLARE var type := value -- variable declaration
  • var := expr -- variable assignment
  • IF cond THEN ... ELSIF cond THEN ... ELSE ... END IF
  • FOR var IN start..end LOOP ... END LOOP
  • WHILE cond LOOP ... END LOOP
  • RETURN expr
  • RAISE EXCEPTION 'msg'

Features

Partitioning

RANGE LIST HASH

Automatic partition pruning on WHERE clauses. ALTER TABLE ADD/DROP PARTITION.

Encryption

AES-256-GCM transparent data encryption at table and column level. Built-in crypto functions.

Recursive CTEs

WITH RECURSIVE nums AS (
  SELECT 1 AS n
  UNION ALL
  SELECT n+1 FROM nums
  WHERE n < 10
) SELECT * FROM nums;

Graph Queries

MATCH (a:follows)
  -[e:to_id]->(b:people)
RETURN a.from_id, b.name;

BEGIN, COMMIT, ROLLBACK, SAVEPOINT, RELEASE SAVEPOINT. Isolation levels parsed.

Scripting Engines

Lua 5.4 Built-in

Embedded interpreter with db.execute(), db.query(), db.now(), db.user(), db.log().

Optional LLVM JIT: compiles Lua bytecode to native ARM64/x86_64 via ORC LLJIT. Tiered execution with auto-JIT after 3 calls.

JavaScript macOS

JavaScriptCore with built-in FTL JIT. Full db.* API plus db.tables(), db.views(), db.describe(), db.catalog.

PL/SQL Built-in

Procedural SQL interpreter with DECLARE, IF/ELSIF/ELSE, FOR, WHILE, RETURN, RAISE. Variable substitution with SQL-safe quoting.

Architecture

tdb (top-level facade)
 +-- tdb_sql       (lexer, parser, AST, optimizer, executor, type coercion)
 +-- tdb_catalog   (tables, views, matviews, sequences, indexes, partitions)
 +-- tdb_doc       (XPath/XQuery/GraphQL on JSON/XML columns)
 +-- tdb_script    (Lua 5.4, optional LLVM JIT, optional JavaScriptCore)
 |    +-- tdb_lua  (vendored Lua 5.4.7)
 |    +-- tdb_jit  (optional: Lua bytecode -> LLVM IR -> native)
 |    +-- tdb_jsc  (optional: JavaScriptCore via jsengine.mm)
 +-- tdb_persistence (C storage <-> C++ Database bridge)
 +-- tdb_core      (C: page, storage, buffer, WAL, txn, crypto)
 +-- tdb_index     (C: B+Tree, B-Tree, R-Tree, R+Tree, Hash)

Key files: database.h (6600 lines, main executor), parser.cpp (3800 lines), ast.h (870 lines, 53 statement types), catalog.h (schema metadata), executor.h (Value type system with 22 types).
